# Описание проекта

## Назначение

Проект представляет собой веб-приложение для ведения базы знаний по проектам системного интегратора. Его основная задача - быстро находить ранее реализованные функциональности по примерному названию и показывать, в каких проектах они встречались, на каких платформах были выполнены и сколько трудозатрат потребовали.

Приложение помогает аналитикам, архитекторам, руководителям проектов и пресейл-командам использовать накопленный опыт компании при оценке новых запросов заказчиков.

## Предметная область

В базе знаний хранится информация о проектах внедрения и доработки корпоративных систем. Каждый проект описывается заказчиком, отраслью заказчика, платформой, продуктом платформы и набором реализованных функциональностей.

Для каждой функциональности фиксируются показатели:

- трудозатраты на аналитическую проработку;
- трудозатраты на разработку;
- технологический стек или язык программирования;
- трудозатраты на тестирование.

## Основной сценарий

Пользователь вводит примерное название функциональности, например `интеграция с бухгалтерией` или `учет времени операторов`. Приложение ищет похожие записи в базе знаний и выводит таблицу найденных проектов.

В результатах отображаются:

- название проекта;
- заказчик;
- отрасль;
- платформа;
- продукт платформы;
- найденная функциональность;
- трудозатраты на аналитику, разработку и тестирование;
- суммарные трудозатраты;
- стек разработки;
- процент похожести с поисковым запросом.

## Функциональные возможности

- поиск по приблизительному названию функциональности;
- поиск по заказчику через основную строку поиска;
- фильтрация по заказчику, отрасли, платформе, продукту и стеку;
- сортировка результатов по похожести, трудозатратам и названию проекта;
- просмотр всех проектов и их функциональностей;
- добавление новых проектов через форму;
- сохранение пользовательских данных в локальном хранилище браузера;
- импорт и экспорт базы знаний в формате JSON;
- восстановление демонстрационных данных.

## Текущая реализация

Приложение реализовано как Docker-композиция из frontend, backend и базы данных. Frontend раздается через Nginx, backend реализован на Python FastAPI, данные хранятся в PostgreSQL.

Основные файлы проекта:

- `index.html` - структура интерфейса;
- `styles.css` - визуальное оформление и адаптивная верстка;
- `app.js` - работа интерфейса с backend API;
- `backend/` - Python FastAPI, модели, миграции и seed-данные;
- `docker-compose.yml` - запуск frontend, backend и PostgreSQL;
- `README.md` - краткая инструкция по запуску и формат JSON.

## Хранение данных

Данные хранятся в PostgreSQL внутри Docker volume. При первом запуске backend применяет миграции Alembic и загружает демонстрационные данные, если база пуста.

## Возможное развитие

- подключение полноценной базы данных;
- авторизация пользователей и разграничение прав;
- карточка проекта с расширенным описанием;
- справочники отраслей, платформ, продуктов и стеков;
- загрузка данных из Excel;
- версионирование оценок трудозатрат;
- расширенный поиск с морфологией русского языка;
- API для интеграции с корпоративными системами.
